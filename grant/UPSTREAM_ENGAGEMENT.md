# Upstream engagement plan

Engagement starts with the precise questions in `docs/UPSTREAM_QUESTIONS.md`, official-source references, exact versions, and minimal valueless reproducers. The project will ask about the persistent wallet service boundary, supported shareable-state observation, idempotency/unknown effects, abort/expiry semantics, and compatibility negotiation.

Issues will be filed only in the official venue selected by GNU Taler maintainers. Proposed patches will be small, tested, and licensed compatibly; upstream is free to decline them. LibreReward will record URLs, decisions, versions, and downstream consequences without representing silence as approval.

Expected outputs include a decision log, compatibility table, reproducer, documentation fixes and/or code patches where requested, and a final report of accepted, pending, and rejected proposals. Initial contact identity and any prior relationship: **[OWNER INPUT REQUIRED]**.
