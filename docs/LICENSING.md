# Licensing and provenance

LibreReward Bridge is offered under AGPL-3.0-or-later. LibreReward may execute GPLv3+ GNU Taler wallet-core/CLI as a separate program; no upstream source is copied or linked into this tree. AGPL remains GPLv3-compatible and also requires network-service source availability, which supports the public-infrastructure goal.

Runtime NPM dependencies in the current lockfile are expected to be permissive. CI runs a production dependency allowlist, vulnerability audit, and CycloneDX SBOM. The optional CLI remains governed by upstream GPLv3+ terms. GNU Taler documentation references may be GPLv3+ or GFDL 1.3+ and are cited rather than copied.

The complete canonical AGPL-3.0 text is present and GitHub detects the repository license. Copyright and AI-assisted provenance are described in `COPYRIGHT.md` and `docs/GENAI_USAGE.md`. Before submitting a funding application, the maintainer must confirm ownership or relicensing authority for every original file and disclose any patent situation; repository automation cannot make that factual declaration on the maintainer's behalf.

This is an engineering record, not legal advice. A future separately distributed SDK may use another FOSS license only after a documented compatibility and provenance review; the current SDK is part of the AGPL-covered repository.
